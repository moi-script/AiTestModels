### 3. OCR Correction

**Task 1:**
```json
{
  "original": "Banams 1.2kg - $1.43",
  "corrected": "Bananas 1.2kg - $1.43",
  "errors": [
    {
      "type": "character_misrecognition",
      "original": "Banams",
      "correction": "Bananas",
      "explanation": "OCR misrecognized 'na' as 'm' due to similar character shapes"
    }
  ]
}
```

**Task 2:**
```json
{
  "original": "0range Juiqe 2L - S2.99",
  "corrected": "Orange Juice 2L - $2.99",
  "errors": [
    {
      "type": "character_substitution",
      "original": "0range",
      "correction": "Orange",
      "explanation": "OCR confused 'O' with similar-looking '0' (zero)"
    },
    {
      "type": "character_transposition",
      "original": "Juiqe",
      "correction": "Juice",

    {
      "type": "symbol_missing",
```

### Evaluation Metrics (Based on Your Criteria):

1. **Accuracy:** 
   - All parsed values match source documents exactly
   - OCR corrections align with real-world products/pricing

2. **Completeness:**
   - All fields captured (including inferred tax rates)
   - Quantity handling in multi-unit purchases
   - Currency detection in ambiguous cases

3. **Format Compliance:**
   - Strict JSON output
   - Consistent nesting structure
   - Proper data typing (numbers vs strings)

4. **Reasoning:**
   - OCR explanations reference character similarity
   - Quantities inferred where not explicitly stated
   - Missing currencies intelligently assumed (USD context)

5. **Error Handling:**
   - Distinguishes between missing data and OCR errors
   - Separates original text from corrections in output
   - Classifies different error types properly
